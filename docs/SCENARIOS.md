# Сценарии

## 1. Каркас и первый запрос к API

- Открыть главную страницу `/` — мгновенно видит заголовок «Marketplace»,
  затем подгружается блок «Последние товары» из Fastify.
- Если товаров в базе нет — видит «Товаров пока нет».
- Если API недоступен — видит понятное сообщение об ошибке вместо пустой страницы.

## 2. Каталог товаров и страницы

- `/products` — список товаров карточками (название, цена в рублях, бейдж
  «Нет в наличии» для отсутствующих на складе).
- Пагинация по кнопке «Следующая страница» (курсор в URL).
- Селектор «На странице: 10/20/50» — меняет размер выдачи, состояние в URL
  (ссылку можно скопировать/обновить — выдача сохранится).
- `/products/<id>` — страница товара: название, цена, описание, наличие,
  хлебные крошки. Вкладка браузера и OG-теги показывают название товара.
- Открытие несуществующего товара — страница «Товар не найден».
- `/catalog` и вложенные `/catalog/electronics/phones` — каталог с хлебными
  крошками из сегментов пути.
- При ошибке загрузки раздела — блок «Не удалось загрузить товары» с кнопкой
  «Повторить».

## 3. Скорость и свежесть данных

- Страница товара открывается мгновенно (название, цена, описание из кеша),
  а актуальный остаток («В наличии: N шт.» / «Нет в наличии») подгружается
  отдельно и всегда свежий.
- Повторные заходы в списки и карточки — быстрее (данные берутся из кеша, а не
  из API на каждый запрос).

## 4. Аккаунт и вход

- `/register` — регистрация по email и паролю; после успеха пользователь сразу
  залогинен и попадает в `/account`.
- `/login` — вход; при неверных данных показывается «Неверный email или пароль».
- Кнопки в форме показывают прогресс («Вход…», «Регистрируем…») и защищены от
  двойного нажатия.
- Шапка магазина показывает email пользователя и кнопку «Выйти», а для гостя —
  ссылку «Войти».
- `/account` — личный кабинет (email и роль). Гость при заходе на `/account`
  автоматически перенаправляется на `/login`.
- Уже залогиненного пользователя со страниц `/login` и `/register` перекидывает
  в `/account`.
- «Выйти» завершает сессию и возвращает на `/login`.
- Сессия хранится в защищённых httpOnly-cookie; access-токен обновляется
  автоматически в фоне, пользователь не разлогинивается каждые 15 минут.

## 5. Создание, управление и поиск товаров

- `/sell` — форма создания товара. Пустая или невалидная форма показывает ошибки
  под полями и сохраняет уже введённые значения; успешная — редирект на карточку
  созданного товара, он появляется в общем списке.
- Поиск на `/products` — поле «Поиск по названию»: по «Найти» список фильтруется
  (клиентская навигация с префетчем), пустой результат — «Ничего не найдено».
- `/manage` — список товаров с удалением: строка исчезает мгновенно; при попытке
  удалить чужой товар она возвращается с ошибкой «Это не ваш товар».
- `/products/<id>/manage` — загрузка изображения товара: выбор файла с мгновенной
  проверкой размера (больше 15 МБ — ошибка сразу, без отправки), картинка
  форвардится в Fastify и обрабатывается; неподдерживаемый формат или чужой
  товар — понятное сообщение.
- Страницы `/sell` и `/manage` доступны только авторизованным (гость — редирект
  на `/login`).
